# 硬件与电气边界

项目源码只明确 DHT22 与 BH1750 的软件接口。下表是源码与现有文档推导的接线边界，不是原理图、PCB 设计或实物已复测的接线证明。

| 模块/信号 | Raspberry Pi 接口 | 当前边界 |
| :-- | :-- | :-- |
| DHT22 DATA | GPIO4 / BCM 4 / 物理 Pin 7 | 3.3 V 逻辑；DATA 通常需 4.7 kΩ–10 kΩ 上拉，具体以实际模块为准 |
| DHT22 VCC / GND | 3.3 V / GND | 不要把裸传感器 DATA 接到超过 Pi GPIO 允许电平的电路 |
| BH1750 SDA | GPIO2 / I²C1 SDA / 物理 Pin 3 | 常见地址 `0x23` 或 `0x5C`，须以实物扫描结果为准 |
| BH1750 SCL | GPIO3 / I²C1 SCL / 物理 Pin 5 | 使用 3.3 V I²C；确认模块上拉、电源和共地 |

首次接线或改线前应断电。确认树莓派型号、供电能力、模块版本、电压、电平、上拉和公共地；不要将 5 V 信号直接接到 Raspberry Pi GPIO。

本仓不公开继电器、风扇、泵、蜂鸣器或其他执行器方案。没有真实物料、照片和复测证据时，不创建虚构的硬件接线、PCB 或状态图。
