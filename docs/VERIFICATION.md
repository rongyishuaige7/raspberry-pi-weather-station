# 验证说明

## CI 可验证范围

每个默认分支提交执行：

1. 公开文件与凭据边界检查；
2. Python 源码语法与测试；
3. 依赖锁定后的 Flask API 测试；
4. 隔离 MySQL 容器启动、schema 初始化、mock 采集、认证和 API 冒烟；
5. .NET 8 / Avalonia 桌面端还原与构建。

GitHub Actions Artifact 仅保留 14 天，不是永久下载或产品发布包。

## 真机复测仍需完成

要把状态升级为“当前真机已复测”，至少记录测试日期、完整 Git commit、树莓派型号/系统、供电、DHT22、BH1750、I²C 地址、接线与共地，并逐项验证：

- Raspberry Pi 启动和 Python 依赖安装；
- DHT22 的合理温湿度读取与失败路径；
- BH1750 的 I²C 地址、光照变化与失败路径；
- MySQL、采集器、API 与桌面端的真实局域网连接；
- 登录、数据写入、历史查询、阈值设置和报警记录；
- 至少 30–60 分钟受控运行、重启与网络异常观察。

日志、截图和视频必须去除用户名、口令、Token、私网 IP、MAC、地址、EXIF/GPS 和真实个人/场所信息。真实照片或视频存在后再添加公开入口，不能创建空目录或假素材。
